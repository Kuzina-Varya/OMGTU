# РГР: Туристическая фирма

## Описание

Проект реализует реляционную базу данных туристической фирмы. База данных хранит клиентов, страны, особенности климата, отели, варианты путевок, фиксированные скидки и факты продажи путевок.

Стоимость продажи рассчитывается с учетом цены путевки и скидки. Путевки продаются на одну, две или четыре недели.

## Структура базы данных

- `client` — клиенты;
- `climate` — типы климата;
- `country` — страны;
- `hotel` — отели;
- `travel_package` — варианты путевок;
- `discount` — фиксированные скидки;
- `orders` — продажи путевок.

## Примечание о суммировании скидок

В предоставленной физической модели один заказ содержит одну ссылку на скидку. Чтобы сохранить исходную схему, суммируемые фиксированные скидки представлены отдельными комбинированными записями справочника `discount`, например `Постоянный клиент + раннее бронирование`.

## Запуск

1. Создать базу данных PostgreSQL:

```sql
CREATE DATABASE travel_agency_db;
```

2. Создать и активировать виртуальное окружение:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

3. Установить зависимости:

```powershell
pip install -r requirements.txt
```

4. Запустить JupyterLab:

```powershell
python -m jupyter lab
```

5. Открыть `tourism_agency_rgr.ipynb`, при необходимости изменить пароль PostgreSQL и выполнить все ячейки сверху вниз.

Для проверки полного запуска используйте:

```text
Kernel → Restart Kernel and Run All Cells
```
